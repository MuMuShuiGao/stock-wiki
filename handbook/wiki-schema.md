# Wiki 页面 Schema

每个 `.md` 文件以 YAML frontmatter 开头，用 `---` 包裹。**绝不**将 frontmatter 包裹在 ````yaml` 代码块中——直接输出裸的 `---`。

## 共有必填字段

```yaml
---
schema_version: 1           # schema 版本号
title: "贵州茅台"            # 页面标题，必须与文件名（不含 .md）一致
type: "股票"                 # 类型: 股票 | 概念 | 模式 | 市场环境 | 总结
summary: "白酒龙头..."       # 50-120字概述，用于检索召回，严禁照搬正文
created: 2026-06-22 10:30:00  # 创建时间，格式 YYYY-MM-DD HH:mm:ss
updated: 2026-06-22 10:30:00  # 最后更新时间，格式 YYYY-MM-DD HH:mm:ss
last_reviewed: 2026-06-22 10:30:00  # 最后审核时间
---
```

## 共有可选字段

```yaml
aliases:                     # 别名列表
  - "茅台"
tags:                        # 标签
  - "白酒"
  - "消费"
related:                     # 关联页面（wikilink 格式）
  - "[[概念/消费升级]]"
  - "[[股票/五粮液]]"
sources:                     # 原始资料来源（文件名，不带 .md 后缀）
  - "2024年报分析"
```

## 股票私有字段

```yaml
code: "600519"               # 股票代码（必填）
industry: "白酒"              # 所属行业
concepts:                     # 关联概念列表
  - "消费升级"
  - "白马股"
```

## 概念私有字段

```yaml
parent: "消费行业"            # 父概念
catalysts:                    # 催化事件列表
  - "春节消费旺季"
  - "提价预期"
```

## 格式规则

- 不得包含重复字段名
- type 等有中文枚举值的字段，**必须输出中文值**，不得使用英文
- 时间字段格式必须为 `YYYY-MM-DD HH:mm:ss`（包含秒），仅 `YYYY-MM-DD` 为无效格式
